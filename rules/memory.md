# rules/memory.md

## Daily notes

Write to `memory/YYYY-MM-DD.md` after every session (local-only, never committed). Log what you did, decisions made, blockers, and anything future-you needs to know.

## Long-term memory

`MEMORY.md` holds curated context: lessons learned, recurring patterns, architectural decisions. Distill from daily notes; remove outdated entries. Public document — never write secrets, tokens, or raw conversation dumps. Only update during solo sessions. Changes go through PRs per [change-control.md](change-control.md).

## Friday cadence

After the weekly report (ops-report): read the week's daily notes → distill into `MEMORY.md` → remove outdated entries → open a PR.
